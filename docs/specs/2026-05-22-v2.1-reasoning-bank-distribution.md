## Problem Statement

After v2.0.0 the ReasoningBank is functional but limited in three ways that block real adoption:

1. **Retrieval is pure dense (mpnet embeddings).** It fails on exact-token queries вҖ” specific file paths, command names, error codes, library names. A lesson titled "audit_history.jsonl freshness" won't surface for the literal query "audit_history". Hybrid BM25 + dense lifts recall ~15вҖ“25% on technical-document corpora (BEIR benchmarks).

2. **Trajectory ingest is dead code.** `reasoning_bank.py::ingest_trajectories()` exists, but no producer writes `.memory/session_trajectories.jsonl` (file is empty) and `finalize_session.py::ingest_reasoning_bank()` only calls `ingest_lessons`. The Self-Improvement Loop can't learn from session outcomes (success/failure correlations).

3. **Downstream consumers have no update path.** `TEMPLATE_README.md` В§"Keeping the template up to date" instructs users to manually diff `.claude/` вҖ” unsustainable. A small `update_ecosystem` command would let downstream projects pull the AI-infra layer without touching project code.

## Scope

### In scope

- **`scripts/reasoning_bank.py`** вҖ” add BM25 sparse ranking, Reciprocal Rank Fusion, `--mode {dense,sparse,hybrid}` CLI flag (default: `hybrid`). Sparse mode must work without ChromaDB installed.
- **`scripts/finalize_session.py`** вҖ” `record_session_trajectory()` writes one JSONL row per finalize; `ingest_reasoning_bank()` now triggers **both** `ingest_lessons` and `ingest_trajectories` (separate bounded subprocesses, separate `audit_history` rows).
- **`scripts/update_ecosystem.py` (new)** вҖ” cross-platform Python script that diffs and syncs `.claude/`, `.agents/`, `scripts/` from upstream template into a downstream project. `--dry-run` default, `--apply` to commit changes, sha-protection for hand-edited files, never touches `.memory/` or project code.
- **`pyproject.toml`** вҖ” add `[project.optional-dependencies] reasoning = ["rank-bm25>=0.2", "chromadb>=0.5", "sentence-transformers>=3.0"]`. Lazy-import everywhere.
- **CHANGELOG, plugin.json, .ecosystem.toml** вҖ” version bump to v2.1.0.
- **Tests** вҖ” a minimal regression harness (`scripts/test_reasoning_bank.py` or inline `if __name__`) with a fixed query set proving exact-token + paraphrase retrieval both work.

### Out of scope

- Switching embedding model from mpnet.
- ChromaDB schema migration (existing collections stay).
- Web UI / dashboard for ReasoningBank.
- Multi-user / multi-project shared ReasoningBank.
- Auto-ingest of PRs / issues / chat history.
- npm packaging (`npx ...`) вҖ” see Open Questions В§1.

## Proposed Solution

### Interface (CLI)

```bash
# Unchanged
python scripts/reasoning_bank.py ingest_lessons
python scripts/reasoning_bank.py stats

# Now functional (was dead code)
python scripts/reasoning_bank.py ingest_trajectories
python scripts/reasoning_bank.py ingest_all

# Extended retrieve
python scripts/reasoning_bank.py retrieve "query" --top-k 5                # default: hybrid
python scripts/reasoning_bank.py retrieve "query" --mode dense              # legacy behavior
python scripts/reasoning_bank.py retrieve "query" --mode sparse             # BM25 only (no ChromaDB needed)
python scripts/reasoning_bank.py retrieve "query" --mode hybrid --rrf-k 60  # tunable fusion

# New distribution surface
python scripts/update_ecosystem.py --from <git-url-or-path>                 # dry-run by default
python scripts/update_ecosystem.py --from <upstream> --apply
python scripts/update_ecosystem.py --from <upstream> --apply --exclude='.claude/skills'
```

**New runtime dependency:** `rank-bm25` (~5 KB, MIT, single-file). Lazy-imported.

### Data Flow

**Hybrid retrieve:**
1. Compute dense top-20 via ChromaDB (if installed).
2. Compute sparse top-20 via BM25 over `lessons.md` corpus (re-parsed at query time; cached on file mtime via `functools.lru_cache`).
3. Reciprocal Rank Fusion: `score(d) = ОЈ 1/(k + rank_i(d))`, default `k=60`.
4. Return top-K (caller-specified).
5. Sparse-only mode skips step 1 вҶ’ works without ChromaDB.
6. Each retrieve logged to `.memory/retrieval_logs.jsonl` with `mode` field.

**Trajectory ingest:**
1. `finalize_session.py` writes `{date, commit_msg, outcome, task_completion, files_touched, duration_min}` to `session_trajectories.jsonl` (append-only).
2. `ingest_reasoning_bank()` now invokes `ingest_lessons` **and** `ingest_trajectories` as separate bounded subprocesses (`timeout=30 each`, status mapped to `ok | skipped | timeout | error`).
3. Each writes its own `audit_history.jsonl` row (`event: "reasoning_bank_ingest_lessons"`, `..._trajectories`).

**Downstream update:**
1. User runs `python scripts/update_ecosystem.py --from <git-url>` from a downstream project root.
2. Script clones upstream into a temp dir, computes per-file diff for `.claude/`, `.agents/`, `scripts/`, root hooks/configs.
3. Prints sync plan; with `--apply`, copies changed files. Skips `.memory/`, `.env*`, `task.md`. SHA-protects `CLAUDE.md`/`AGENTS.md` if hand-edited (sha mismatch vs. last `.ecosystem.toml` snapshot вҶ’ require `--force` to overwrite).
4. Updates `.ecosystem.toml::version` and `.ecosystem.toml::upstream_sha`.

## Acceptance Criteria

- [ ] `retrieve --mode hybrid` returns results that include both exact-keyword (BM25-favored) and paraphrase (dense-favored) matches on a documented test query set.
- [ ] `retrieve --mode sparse` works **without** chromadb installed (degrade-gracefully fallback path).
- [ ] `finalize_session.py` writes one row to `session_trajectories.jsonl` per finalize and triggers both ingests; both statuses logged to `audit_history.jsonl`.
- [ ] `update_ecosystem.py --dry-run` against this repo as upstream shows zero diff (idempotent against self).
- [ ] `update_ecosystem.py --apply` against a downstream test dir updates `.claude/` and `.agents/` without touching `.memory/` or project files; sha-protected files require `--force`.
- [ ] `rank-bm25` added to `[project.optional-dependencies] reasoning`; missing dep produces a friendly error, not a stack trace.
- [ ] CHANGELOG v2.1 section written; version bumped in `plugin.json` and `.ecosystem.toml`.

## Risks & Mitigations

| Risk | Prob | Mitigation |
|---|---|---|
| BM25 corpus rebuild on every query slow | Low | <500 lessons in practice; `lru_cache` keyed on `lessons.md` mtime |
| RRF `k=60` wrong for this corpus | Medium | Expose `--rrf-k`; document tuning notes in `template-design.md` |
| `update_ecosystem.py` clobbers user edits | **High** | Default `--dry-run`; require `--apply`; sha-check protected files; never touch `.memory/`; `--force` gate for overrides |
| Trajectory ingest balloons ChromaDB size | Low | Rotation at 1000 rows in `prune_memory.py` (already exists) |
| Adding `rank-bm25` breaks current envs | Low | Lazy-import; sparse mode requires it, dense/hybrid degrade if absent |
| Hybrid default changes /new_session retrieve behavior subtly | Medium | A/B compare on existing query log before flipping default; document in CHANGELOG |

## Resolved Decisions (was: Open Questions)

1. **Distribution:** ship as `scripts/update_ecosystem.py` (Python, zero-install, lives in the template repo). PyPI/npx packaging deferred to v2.2 if real downstream demand emerges.
2. **Trajectory schema вҖ” expanded:** `{date, commit_msg, outcome, task_completion, files_touched, duration_min, tools_used}`. `parse_trajectories()` will be updated to read all fields; `document` string keeps the same 3-field summary for embedding.
3. **`retrieve` default = `--mode hybrid`.** Existing `/new_session` callers (no `--mode`) gain hybrid behaviour automatically; backward-compat-sensitive callers can pin `--mode dense`.
4. **Sparse-only fallback shipped.** `--mode sparse` works without ChromaDB installed. Code path: separate BM25 retrieve function with no chromadb imports; hybrid/dense paths lazy-import as today.
