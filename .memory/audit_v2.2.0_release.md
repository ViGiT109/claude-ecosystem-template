## 4. Lesson Candidates для `.memory/lessons.md`

### Lesson 1: **pre-commit framework silently passes language=system hooks** (NEW, P1)
- **Симптом:** hook entry с `language: system` + `stages: [pre-push]` показывает "Passed" за 0.07s, не выполняя реально target скрипт.
- **Проверочный сценарий:** инжект `SyntaxError` в hook script → pre-commit всё равно "Passed" (но direct call exit=1).
- **Promotion:** прецедент новый → начать с lesson; если повторится — переписать пре-push hook как raw `.git/hooks/pre-push` script (bypass framework).
- **Связь с прошлым lesson «ship-prod-run-before-DONE»:** мы тестировали через direct call (`python scripts/check_version_sync.py --pre-push <<< stdin`), не через **реальный** `git push --tags`. Lesson v2.1.1 требует prod run перед DONE — он формально соблюдён только частично: standalone test был, но end-to-end pipeline test (через `git push`) — нет.

## 6. Известный P1 баг (главный)

### Описание
Pre-commit framework на этой Windows-машине рапортует "Passed" не запуская hook script. Это сломанная главная фича Layer A (deterministic version-sync guardrail для git push --tags).

### Доказательство (независимое подтверждение)
1. `echo "refs/tags/v9.9.9 deadbeef refs/tags/v9.9.9 0000000" | pre-commit run --hook-stage pre-push check-version-sync --verbose` → "Passed (0.07s)".
2. `python scripts/check_version_sync.py --pre-push <<< "refs/tags/v9.9.9 deadbeef refs/tags/v9.9.9 0000000"` → exit=1 с полной диагностикой.
3. Duration 0.07-0.08s **физически невозможна** для Python interpreter startup + import json/re/pathlib + 4 file reads — это classic silent-skip.

### Серьёзность
**P0 для функциональности Layer A.** Spec explicitly обещает (line "блокирует git push --tags, если версия рассинхронизирована"). Обещание не выполняется.

### Mitigation сейчас
- Standalone CLI работает корректно — можно вызывать manually перед каждым `git push --tags`.
- Прецедент в lessons.md «single-source-of-truth-for-version» уже учит CHANGELOG+plugin.json+tag синхронность держать руками — обходной путь существует.

### Решение для v2.2.1
**Bypass pre-commit framework для этого хука.** Установить raw `.git/hooks/pre-push` (с шиммом, который сначала вызывает `python scripts/check_version_sync.py --pre-push` напрямую, потом делегирует в pre-commit's hook-impl для остальных pre-push hooks, если будут). Это снимает зависимость от silent-skip баги framework.

Альтернатива (если есть время investigate): убрать `language: system`, попробовать `language: python` с явным `entry: scripts/check_version_sync.py` — может быть, дело в `system` + Windows shlex.
