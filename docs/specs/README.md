# Specifications

> Specs are required for tasks > 2 hours or > 3 files.
> Use `/create_spec` to generate a new spec from the template.

## Naming convention

`YYYY-MM-DD-short-description.md`

Example: `2026-05-17-user-authentication.md`

## Process

1. Run `/create_spec` → fills the spec template.
2. Review spec with the user → get approval.
3. Create `task.md` from the Acceptance Criteria.
4. Begin implementation only after approval.
