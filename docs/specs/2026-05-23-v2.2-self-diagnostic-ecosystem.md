## Problem Statement

После v2.1.1 экосистема даёт надёжные **детерминистические гарантии** (хуки, pre-commit guardrail), но `/audit_ecosystem` остаётся **user-triggered**. Никто не запускает диагностику автоматически. Также нет ни дашборда метрик, ни проверки самих хуков, ни механизма предупреждения о version drift до тега.

Конкретные слепые зоны:

1. **Нет авто-триггера для аудита.** Если юзер не попросит `/audit_ecosystem` — диагностика не случится. Score дрейфит релиз за релизом, никто не видит.
2. **Нет тестов самих хуков.** Если `session_start.py` упадёт после апгрейда Python — узнаем по тишине наверху новой сессии.
3. **Нет дашборда метрик.** Журналы (`audit_history.jsonl`, `retrieval_logs.jsonl`, `session_trajectories.jsonl`) пишутся, но никто их не агрегирует.
4. **Version drift всплывает в пост-аудите, а не на push.** v2.1.0 audit нашёл, что `.ecosystem.toml::version` отстал от `plugin.json` — уже после тега. Lesson записан, но детерминированной проверки нет.
5. **`lessons.md` растёт, не уплотняется.** Скилл `consolidate-memory` существует, но триггера нет.

Юзер сформулировал цель: «самодостаточная, эволюционирующая, саморазвивающаяся, **самодиагностирующаяся** AI-экосистема».

## Scope

### In scope

Три слоя в порядке силы гарантий:

- **Layer A — Deterministic guardrails:**
  - `scripts/check_version_sync.py` — pre-push hook, проверяет синхронность `plugin.json::version` ↔ последнее `## [X.Y.Z]` в CHANGELOG ↔ имя push'имого тега ↔ `.ecosystem.toml::ecosystem.version` (если есть).
  - Регистрация через `.pre-commit-config.yaml::stages: [push]`.
  - `scripts/check_hook_health.py` — детерминистическая проверка, что все хуки в `.claude/settings.json` существуют и не падают на синтетическом stdin.

- **Layer B — Session-triggered cron:**
  - Триггер — открытие новой сессии (`session_start.py`). Не OS-level cron, не Anthropic-hosted scheduled-tasks — портируется в любой downstream-проект.
  - В `session_start.py` расширяем существующие freshness-сигналы + добавляем новые: auto-audit overdue, consolidate-memory overdue, hook health stale.
  - При просрочке — эмитим `🔴 X REQUIRED` или `🟡 X RECOMMENDED` маркер в инжект. Агент видит → запускает работу → пишет в журнал → долг сбрасывается.

- **Layer C — Observability:**
  - `scripts/diag_dashboard.py` — один прогон, читает все журналы, печатает markdown-сводку (audits, lessons, retrieval mix, trajectories, hooks, version sync). `--summary` флаг — 5 строк светофоров.
  - `.claude/commands/diag_status.md` — slash-команда, вызывает дашборд `--summary`.
  - Расширение SessionStart инжекта — единый блок `## 📊 Ecosystem health` (объединяет audit/lessons/hooks/version-sync индикаторы).

- **`.claude/commands/consolidate_lessons.md` (new)** — slash-команда, триггерит `anthropic-skills:consolidate-memory` и после успеха пишет `consolidate_complete` row в `audit_history.jsonl`.

- **`.claude/hooks/_ecosystem_health.py` (new)** — общий stdlib-модуль (read journals, compute freshness, format markers). Используется из `session_start.py`, `stop_audit.py`, `diag_dashboard.py`.

- **CHANGELOG, plugin.json, `.ecosystem.toml`** — bump 2.1.1 → 2.2.0.

- **`task.md`** для спринта v2.2 — с явным разделом Phase Release (lesson «release-checkboxes-in-task-md» применяется заранее, не задним числом).

### Out of scope (намеренно)

- **OS-level cron / Task Scheduler / Anthropic scheduled-tasks MCP** — портируемость в downstream важнее. Session-triggered cron самодостаточен.
- **Доступ к сырым транскриптам прошлых сессий** — артефактов в `.memory/` + git log достаточно (доказано v2.1.0/v2.1.1 практикой).
- **Внешние каналы алёртов (email/Slack/push)** — observability ограничена сессией Claude Code.
- **Lesson auto-promotion → rule** — ручной upgrade path остаётся в CLAUDE.md. Семантический анализ — кандидат в v2.3.
- **Tools usage tracking** в `session_trajectories.jsonl::tools_used` — остаётся placeholder'ом, wire-up требует in-process aggregator'а.

## Phases

**Phase 1 — Pre-push version-sync guardrail (Layer A).** `scripts/check_version_sync.py` + регистрация в `.pre-commit-config.yaml` + опт-ин в bootstrap.

**Phase 2 — Hook health-check (Layer A + B).** `scripts/check_hook_health.py` + раздел «Hooks» в `scripts/health_check.py` + опциональный dry-run mode в каждом хуке.

**Phase 3 — Auto-audit trigger в session_start.py (Layer B).** Расширение `emit_audit_freshness()`: age > 7 дней ИЛИ stop_hook count > 20 → 🔴 `AUDIT REQUIRED`. Окно для инкрементального аудита — `<last_audit_tag>..HEAD`. Введение модуля `_ecosystem_health.py`.

**Phase 4 — Consolidate-memory trigger (Layer B).** `emit_consolidate_freshness()` симметрично audit. `/consolidate_lessons` slash-команда пишет `consolidate_complete` event.

**Phase 5 — Diagnostic dashboard + `/diag_status` (Layer C).** `scripts/diag_dashboard.py` + slash-команда. Шесть секций: audits, lessons, retrieval, trajectories, hooks, version-sync.

**Phase 6 — Ecosystem health injection в session_start.py (Layer C).** Единый блок `## 📊 Ecosystem health` в инжекте, объединяющий разрозненные сигналы.

**Phase 7 — Release v2.2.0.** Bump, CHANGELOG, тег, **самотест pre-push hook'ом** (намеренно расходим версии, проверяем что блокирует, чиним, push идёт), post-release аудит ≥ 85/100. Если < 85 — автономный хотфикс v2.2.1.

## Triggers and thresholds

| Долг | Триггер | Маркер | Куда смотреть |
|---|---|---|---|
| Audit overdue | age > 7 дней ИЛИ stop_hook count > 20 с последнего `audit_complete` | 🔴 `AUDIT REQUIRED` | `audit_history.jsonl::event=="audit_complete"` |
| Audit aging | age 3–7 дней | 🟡 `audit aging` | то же |
| Consolidate overdue | age > 30 дней ИЛИ lesson count > 20 | 🟡 `CONSOLIDATE RECOMMENDED` | `audit_history.jsonl::event=="consolidate_complete"` |
| Hook health stale | age > 24h | 🟡 `HOOK CHECK DUE` | `audit_history.jsonl::event=="hook_health_check"` |
| Version drift | при `git push --tags` | EXIT 1 | детерминистическая проверка |

Все пороги — параметризуемы через `.ecosystem.toml::[ecosystem.thresholds]` (новая опциональная секция). По умолчанию консервативные значения выше.

## Verification

**Unit-level (по фазам):** см. одобренный план `~/.claude/plans/magical-drifting-reddy.md` §Verification.

**End-to-end (релизный поток):**

1. `git push --tags v2.2.0` → pre-push guardrail проверяет → push идёт.
2. Намеренно расходим `.ecosystem.toml::version` → push блокируется. Поправить → push проходит.
3. Открыть новую сессию → инжект показывает 🔴 `AUDIT REQUIRED` (только что был релиз).
4. Запустить `ecosystem-auditor` сабагент с окном `v2.1.1..v2.2.0` → row в журнал → следующая сессия 🟢 audit.
5. `/diag_status` → все индикаторы 🟢.

**Кросс-платформенность:** Windows + Linux/macOS — все скрипты на stdlib Python, pre-commit framework агностичен.

**Целевой балл аудита:** ≥ 85/100 (🟢). Если ниже — автономный хотфикс v2.2.1.

## Risk register

| Риск | Митигация |
|---|---|
| Pre-push hook ломает flow юзера на других машинах | Опт-ин через `pre-commit install --hook-type pre-push`. Снимается `pre-commit uninstall --hook-type pre-push`. |
| `_ecosystem_health.py` импортируется хуками вне `.venv` | Чистый stdlib (json, pathlib, datetime, os, re). |
| Авто-аудит слишком частый и надоедает | Пороги в `.ecosystem.toml::[ecosystem.thresholds]`. По умолчанию консервативные. |
| `_run.py` не найдёт Python без `.venv` | Fallback на `python3` / `python` в PATH уже работает. |
| Hook health-check сам ломается тихо | Если `hook_health_check` event старше 48h — `session_start.py` эмитит 🔴 (рекурсивная защита). |
